# Aula 19 - Revisão e boas práticas

**Data:** 23/11/2026

---

## Apresentação

Organização de pastas (screens, components, services/db); nomenclatura consistente; ciclo de vida e fluxo de dados. README com telas, tabelas e como rodar.

---

## Slides

### Objetivo

Revisar organização do código e documentar a estrutura do projeto para facilitar a correção e a apresentação final.

### Pastas

screens/ (telas), components/ (reutilizáveis), contexto/, services/ ou db/ (banco). constantes.js para chaves e cores.

### Nomenclatura

Telas: sufixo Screen. Funções: verbo + nome (carregarItens, salvarNoBanco). Constantes: UPPER_SNAKE_CASE.

### README

Nome do app e tipo (A, B, C ou D); lista das telas e o que fazem; tabelas SQLite (nome e colunas); comandos para rodar (npm install, npx expo start).

### Atividade da quinzena

Revisar projeto em andamento; criar README com estrutura das telas, do banco e instruções para rodar.

### Próxima aula

Preparação do projeto final: escopo da Fase 4, checklist de entrega e dicas de apresentação.
