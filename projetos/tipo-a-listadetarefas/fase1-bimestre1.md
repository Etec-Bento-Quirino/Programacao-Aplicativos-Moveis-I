# Tipo A – Fase 1 (Bimestre 1)

**Sugestão de execução:** quinzena 6 (13/04/2026 a 16/04/2026).

## Objetivo

Entregar um app com duas telas: uma lista estática de tarefas e uma tela de detalhe ao tocar em um item. Incluir um botão “Adicionar” (pode não salvar ainda).

## Requisitos

1. **Tela Lista**
   - Lista com pelo menos 3 tarefas fixas (ex.: título + “Pendente”).
   - Ao tocar em um item, navegar para a tela de detalhe passando o título (ou id).
2. **Tela Detalhe**
   - Exibir título e descrição da tarefa (texto fixo ou recebido por parâmetro).
   - Botão “Voltar” para retornar à lista.
3. **Navegação**
   - React Navigation (stack): Lista ↔ Detalhe.
4. **Botão “Adicionar”**
   - Na tela lista, botão que pode abrir um alert “Em breve” ou uma tela em branco (não é obrigatório salvar na Fase 1).

## Conteúdos cobrados

- Layouts e listas (Aulas 03, 05).
- Navegação entre telas (Aula 06).
- Texto e botões (Aula 04).

## Critérios de avaliação

- Duas telas com navegação funcionando.
- Lista exibindo itens; toque abre o detalhe.
- Código organizado (pastas/screens).

## Dica

Use o tutorial da Aula 05 (listas) e da Aula 06 (navegação) como base. Os dados podem ser um array constante no código.
